## Resolution (2026-07-09)

The reports are generated by this project's recipe system (the scheduled rituals and
the morning routine); each raw ritual dumped unfenced command stdout into a `.md`
file, which is why it read as slop. Fixes, by item:

1 & 2 (markdown formatting / unreadable standup) — every captured-command report now
renders its output inside a fenced code block with a clean Markdown header (H1 title,
`**Generated**` time, code-formatted `**Command**`). Empty output reads as an explicit
"No output." An output that itself contains a fence cannot break out (the fence length
adapts). Single writer: `buildCommandReport` in
[extension/src/exec/actionRunner.ts](../../../../extension/src/exec/actionRunner.ts).

3 (too many docs / consolidate + link) — the morning-routine summary now has a
**Report** column linking each member's own report, relative to the summary file, so
the summary is the one clickable index over the day's sub-reports. Command-action
members (project stats, pubspec freshness) return their written path so they link too.
See [routineRunner.ts](../../../../extension/src/exec/routineRunner.ts) and the command
dispatcher in [actionRunner.ts](../../../../extension/src/exec/actionRunner.ts).

4 (pubspec out-of-date, only stale) — new
[pubspecOutdated.ts](../../../../extension/src/exec/pubspecOutdated.ts) parses
`dart pub outdated --json` and reports ONLY packages behind latest (discontinued flagged
inline); an all-current project gets a one-line all-clear. A pubspec project's
"Dependency freshness" ritual now routes to this filtered command.

5 (lint + TODOs, flutter + saropa lints) — the lint sweep already runs
`flutter analyze && dart run custom_lint` when saropa_lints/custom_lint is configured.
The TODO/tech-debt harvest and dependency freshness are now morning-routine members, so
the daily report covers lint issues, TODO markers, and stale packages. See
[routineRecipes.ts](../../../../extension/src/recipes/routineRecipes.ts) and
[scheduledRecipes.ts](../../../../extension/src/recipes/scheduledRecipes.ts).

Note on scope: the routine membership change updates the DEFAULT the extension seeds
and detects. A project that already stored a Morning routine (its members live in that
project's `.vscode/saropa-workspace.json`) keeps its saved membership until re-seeded or
edited — this project cannot edit another project's config.

New report format convention recorded in section 4.8 of
[plans/guides/STYLEGUIDE.md](../../../guides/STYLEGUIDE.md).

## Finish Report (2026-07-09)

### Defect

Scheduled report rituals captured a command's combined stdout/stderr and wrote it to
a dated Markdown file as `header + rawBody`, with no code fence. A Markdown preview
rendered `git log --stat` / `git status` output as prose, mangling it. The morning
routine wrote a member-outcome summary but did not link the reports its members
produced, leaving several disconnected files with no index. There was no
only-out-of-date dependency report, and the dependency-freshness and TODO/tech-debt
rituals were not part of the morning routine, so the daily output omitted stale
packages, lint issues, and TODO markers.

### Change

- `exec/actionRunner.ts`: added `fenceBlock` (adaptive backtick length) and
  `buildCommandReport`; `runShellToReport` now writes the fenced, headed report and
  an explicit `_No output._` on an empty result. `runVsCommand` forwards the command
  result; the `command` dispatch case records a returned report path so a routine can
  link command-produced reports.
- `exec/routineRunner.ts`: the summary gained a Report column that links each member's
  report relative to the summary file (`reportLink`). Each member's stale report entry
  is cleared before the member runs, so the summary links only a report the current
  run wrote.
- `exec/lastReport.ts`: added `peekLastReport` (non-consuming read) and
  `clearLastReport`.
- `exec/projectStats.ts`: `runProjectStats` returns the written report path so it links
  in the summary.
- `exec/pubspecOutdated.ts` (new): parses `dart pub outdated --json`, filters to only
  packages behind latest (discontinued flagged), writes a compact report, opens it only
  when something is stale, and returns the path. Registered in `activation/wiring.ts`.
- `recipes/scheduledRecipes.ts`: a pubspec project's dependency-freshness ritual routes
  to the filtered command; other ecosystems keep the raw (now-fenced) capture.
- `recipes/routineRecipes.ts`: the tech-debt/TODO harvest and dependency freshness are
  morning-routine members.
- `i18n/locales/en.json`: `deps.*` strings.

### Review outcome

A read-only review found one correctness defect in the new code: because the
per-member report registry was read without clearing, a second routine run in which a
member wrote no report (a failed dependency check) would relink the previous run's
stale, wrong-dated report. Fixed by clearing each member's entry before it runs; a
regression test pins the behavior.

### Verification

`tsc --noEmit` clean; `npm run test:unit` — 920 passing (8 added: command-report
fencing, `isStale` filter, pubspec render, morning order, pubspec routing, summary
links, stale-relink regression). `node esbuild.js` builds the bundle. Device/manual
smoke of a live scheduled run is not covered by the unit suite (see "What to test").

### Scope note

The routine-membership change updates the DEFAULT the extension seeds and detects. A
project that already stored a Morning routine keeps its saved membership (stored in
that project's own `.vscode/saropa-workspace.json`) until re-seeded or edited; this
project cannot modify another project's config.
